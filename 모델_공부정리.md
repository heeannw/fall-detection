# 낙상 감지 프로젝트 - 모델 공부 정리
> MediaPipe는 정리 완료. 이 문서는 **YOLO / VideoMAE / XGBoost** 3가지 정리.

## 1. YOLO (You Only Look Once)

### 개념
YOLO는 이미지에서 물체를 실시간으로 감지(Object Detection)하는 딥러닝 모델이다.
이름처럼 이미지를 **한 번만 보고(one look)** 물체의 위치(bounding box)와 종류(class)를 동시에 예측한다.

기존 방식(R-CNN 계열)은 "후보 영역 찾기 → 분류" 2단계를 거쳤지만, YOLO는 1단계로 처리해서 속도가 훨씬 빠르다.

### 동작 원리
1. 입력 이미지를 격자(grid)로 나눈다.
2. 각 격자 셀이 bounding box 좌표 + confidence + 클래스 확률을 동시에 예측한다.
3. 겹치는 box들을 NMS(Non-Maximum Suppression)로 정리해 최종 결과를 낸다.

### 우리 프로젝트에서의 사용
- **파일**: `detector/yolo_detector.py`
- **모델**: 직접 파인튜닝한 커스텀 모델 (`runs/detect/fall_detection_v3/weights/best.pt`)
  - `train_yolo.py`로 낙상 데이터셋에 재학습시킨 것
- **역할**: 프레임에서 사람이 쓰러진 자세(fall 클래스)를 bounding box로 감지
- **판정 방식**: fall 클래스 confidence가 **0.7 이상**이면 score +1
- **처리 주기**: 매 3프레임마다 1회 실행 (캐싱으로 성능 최적화)

```python
# yolo_detector.py 핵심 로직
if "fall" in label.lower() and conf > 0.7:
    return 1   # fall 감지
return 0
```

- **추가 활용**: XGBoost 학습 시 feature로도 사용
  - fall 클래스의 raw confidence 값(임계값 적용 전)을 53개 특징 중 하나로 포함

### 핵심 포인트 (발표용)
- YOLO = 빠른 실시간 물체 감지 / 우리는 낙상 자세 자체를 직접 인식
- 직접 파인튜닝 → 일반 YOLO가 아닌 낙상 전용 모델
- 한계: 단일 프레임만 봄, 움직임의 흐름(시계열)은 모름

## 2. VideoMAE

### 개념
VideoMAE는 **비디오를 이해하는 자기지도학습(Self-Supervised) 모델**이다.

MAE = Masked Autoencoder. 원래 이미지용으로 개발됐고, VideoMAE는 이를 비디오로 확장했다.
학습 방법: 비디오 패치의 **90%를 무작위로 가린(masking) 뒤 복원**하도록 훈련 → 비디오의 시공간적 패턴을 자연스럽게 학습함.

파인튜닝하면 비디오 분류(행동 인식)에 강력한 성능을 낸다.

### 동작 원리
1. 비디오를 일정 길이의 클립(우리 프로젝트: **16프레임**)으로 자른다.
2. 각 프레임을 패치로 분할하고 시공간 transformer로 처리한다.
3. 전체 클립을 보고 어떤 행동인지 분류한다.

단일 프레임이 아닌 **연속된 프레임의 흐름**을 보기 때문에 "쓰러지는 동작" 자체를 포착할 수 있다.

### 우리 프로젝트에서의 사용
- **파일**: `detector/videomae_detector.py`
- **모델**: HuggingFace 사전학습 모델 파인튜닝 버전
  - `yadvender12/videomae-base-finetuned-kinetics-finetuned-fall-detect`
  - Kinetics 데이터셋으로 사전학습 → 낙상 감지용으로 추가 파인튜닝된 것
- **입력**: 224×224 크기로 리사이즈한 16프레임 버퍼
- **출력 클래스**: FallDown, LyingDown, Sitting, Walking
- **판정 방식**:
  - FallDown + confidence ≥ 0.6 → score +2
  - LyingDown + confidence ≥ 0.85 → score +1
- **처리 방식**: 별도 백그라운드 스레드(`videomae_worker`)에서 0.5초마다 실행

```python
# videomae_detector.py 핵심 로직
if label == "FallDown" and confidence >= 0.6:
    last_score = 2
elif label == "LyingDown" and confidence >= 0.85:
    last_score = 1
else:
    last_score = 0
```

- **왜 임계값이 다른가**: LyingDown은 일상적으로 누워있는 것(폰 보기, 취침)과 구분이 어려워 더 높은 임계값(0.85) 적용

### 핵심 포인트 (발표용)
- VideoMAE = 비디오의 시간적 흐름까지 보는 모델 / YOLO가 못 잡는 "쓰러지는 동작" 포착
- 사전학습 모델 사용 (직접 학습 X) → 빠르게 적용 가능
- 한계: 무겁고 느림 → 10프레임마다 1회 실행, 별도 스레드로 분리

## 3. XGBoost

### 개념
XGBoost = eXtreme Gradient Boosting. **여러 개의 약한 결정 트리(Decision Tree)를 순차적으로 쌓아** 강한 예측기를 만드는 앙상블 학습 방법이다.

앞 트리가 틀린 오류를 다음 트리가 보정하는 방식(gradient boosting)이고, XGBoost는 이를 매우 빠르고 효율적으로 구현한 라이브러리다.

딥러닝이 아닌 **전통적 머신러닝**이지만, 정형 데이터(숫자 feature)에서는 종종 딥러닝보다 뛰어난 성능을 낸다.

### 동작 원리
1. 처음에 단순한 예측값으로 시작한다.
2. 잔차(오류)를 계산하고, 그 오류를 맞추는 새 트리를 추가한다.
3. 이를 n_estimators 횟수만큼 반복한다.
4. 최종 예측 = 모든 트리의 예측을 합산(가중 합).

### 우리 프로젝트에서의 사용 (Phase 2)
- **파일**: `train_phase2_xgboost.py`, `detector/phase2_detector.py`
- **목적**: MediaPipe + YOLO + VideoMAE의 rule-based 판정의 한계를 보완하는 ML 판정기
- **입력 특징 (53개/프레임)**:
  - MediaPipe 핵심 keypoint 13개 × (x, y, visibility) = **39개**
  - 파생 특징: horizontal_ratio, sh_diff, hk_diff, velocity, acceleration = **5개**
  - 자세 one-hot (lying/sitting/standing/unknown) = **4개**
  - YOLO fall raw confidence = **1개**
  - VideoMAE 클래스 확률 (FallDown, LyingDown, Sitting, Walking) = **4개**

- **시계열 윈도우 특징**: 현재 프레임 + 과거 8프레임의 mean/max → **53 × 3 = 159개**로 확장
  - 이렇게 하면 단일 프레임이 아닌 흐름을 반영할 수 있음

- **학습 데이터**: Le2i + uttej 낙상 데이터셋, 영상 단위 80/20 분할
- **클래스 불균형 처리**: `scale_pos_weight`로 낙상 프레임(소수)의 가중치를 높임
- **성능**: F1 Score **94.3**
- **실시간 판정 임계값**: XGBoost 확률 ≥ 0.7 → fall 판정 (`XGBOOST_THRESHOLD`)

```python
# main.py 앙상블 로직
if ENSEMBLE_MODE == "and":
    ensemble_pos = rule_based_pos and xgb_pos   # 최저 오탐율
elif ENSEMBLE_MODE == "or":
    ensemble_pos = rule_based_pos or xgb_pos    # 최고 민감도 (기본값)
elif ENSEMBLE_MODE == "ml_only":
    ensemble_pos = xgb_pos
else:  # rule_only
    ensemble_pos = rule_based_pos
```

### 핵심 포인트 (발표용)
- XGBoost = 숫자 feature를 잘 분류하는 ML 모델 / 나머지 모델들의 출력값을 통합해 최종 판단
- 단일 모델이 아닌 **메타 분류기(Meta-Classifier)** 역할: 다른 모델 결과를 입력으로 받아 더 정확하게 판단
- 영상 단위 학습/평가: 같은 영상 프레임이 train/val에 동시에 들어가지 않도록 영상 단위로 분할

## 전체 시스템 흐름 요약

```
카메라 프레임
    │
    ├── MediaPipe → 관절 각도/자세 분석 → score
    ├── YOLO → 쓰러진 자세 bounding box → score
    ├── VideoMAE → 16프레임 행동 분류 → score
    │
    ├── [rule-based] score 합산 ≥ 임계값 → fall?
    │
    └── XGBoost → 53개 특징 기반 확률 → fall?
           (MediaPipe + YOLO + VideoMAE 출력 모두 포함)
    
앙상블 (or/and/ml_only/rule_only) → 최종 낙상 판정
```

| 모델 | 보는 것 | 역할 | 처리 주기 |
|------|---------|------|-----------|
| MediaPipe | 단일 프레임 관절 | 자세 분류 + 정지 감지 | 2프레임마다 |
| YOLO | 단일 프레임 전체 | 쓰러진 자세 직접 감지 | 3프레임마다 |
| VideoMAE | 16프레임 클립 | 행동 흐름 분류 | 0.5초마다 |
| XGBoost | 53개 수치 특징 | 통합 ML 판정 | 3프레임마다 |

## 자주 나올 수 있는 질문

**Q. 왜 여러 모델을 같이 써?**
A. 각 모델이 잡을 수 있는 낙상 유형이 다르다. YOLO는 자세 형태를, VideoMAE는 동작 흐름을, XGBoost는 종합적 패턴을 본다. 하나만 쓰면 놓치는 경우(오탐, 미탐)가 생겨서 앙상블로 보완.

**Q. XGBoost가 딥러닝보다 나은 이유?**
A. 입력이 이미 처리된 숫자 feature이기 때문. 딥러닝은 raw 데이터에서 특징을 스스로 학습할 때 강하고, 이미 의미있는 숫자로 변환된 데이터에서는 XGBoost가 더 빠르고 해석 가능하다.

**Q. VideoMAE의 masking이 뭐야?**
A. 학습할 때 비디오의 90%를 가리고 나머지만 보고 가려진 부분을 복원하게 훈련. 이 과정에서 "비디오에서 무엇이 중요한지"를 자연스럽게 학습. 파인튜닝만 해도 행동 분류를 잘 하는 이유.

**Q. YOLO를 직접 학습시킨 이유?**
A. 일반 YOLO는 "person", "car" 같은 일반 클래스만 있다. 낙상(fall)이라는 특수 클래스를 인식하려면 낙상 데이터셋으로 재학습(파인튜닝)이 필요.
