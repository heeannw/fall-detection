# 옵션 C — 라이브 자가 테스트 가이드

실제 카메라 앞에서 직접 테스트해서 시스템 신뢰도를 검증합니다.
정지 이미지(A)·녹화 영상(B)에서 측정된 성능이 **실사용 환경에서도 유지되는지** 확인하는 마지막 단계입니다.

## 준비

```
cd C:\fall-detection
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

브라우저에서 `http://localhost:8000/video` 열어 실시간 화면 확인 (또는 프론트엔드).

확인할 화면 정보:
- 좌측 상단 라벨: `Normal` / `FALL DETECTED!` / `ACCIDENT!`
- 자세: `Posture: lying(OK/ABNORMAL) [TOO CLOSE]`
- 점수: `MediaPipe / YOLO / VideoMAE` 각각의 점수
- 임계값: `(N/M pt)` 형식으로 현재 점수 / 자세별 임계값 표시

각 시나리오에서 **체크리스트**의 모든 항목이 통과하면 ✅, 하나라도 실패하면 ❌.

---

## 시나리오 1 — 명확한 낙상 (정면)

**동작**: 카메라 정면에 서서 → 빠르게 옆으로 쓰러져 누움 (안전한 매트 위 권장)

**예상**:
- 자세: `standing` → `lying` (자세 전환 보너스 +2 발동)
- 점수: 4~7pt 도달
- 라벨: `FALL DETECTED!` (0.5초 이내)
- 알림: 알람음 재생, 캡처 저장 (`captures/` 폴더)

**통과 기준**:
- [ ] 쓰러진 후 1초 이내 FALL DETECTED 표시
- [ ] 알람음 재생
- [ ] `captures/` 폴더에 1장 저장됨 (메타데이터 JSON도)
- [ ] 30초 동안 동일 사건으로 추가 캡처 안 됨 (쿨다운 정상)

---

## 시나리오 2 — 명확한 낙상 (측면)

**동작**: 카메라 옆면에서 측면 자세 → 앞으로 쓰러짐

**예상**:
- 자세: `standing` → `lying`
- 가속도 점수 추가 가산 가능 (`velocity > 1.5`)
- 라벨: `FALL DETECTED!`

**통과 기준**:
- [ ] 1초 이내 감지
- [ ] 가속도 기반 점수가 화면에 표시됨 (MediaPipe 점수↑)

---

## 시나리오 3 — 천천히 누움 (정상)

**동작**: 의자에서 일어선 후 → 천천히 5~6초에 걸쳐 매트에 누움 → 1분 정도 그대로 있음

**예상**:
- 자세: `standing` → `sitting` → `lying`
- 자세 전환 보너스 발동 가능 (5초 이내 전환)
- 1분 정지 후에도 → 미세 움직임(호흡)이 있으면 `accident_by_stillness` 발동 X
- 라벨: 일시적으로 FALL이 떴다가 → 시간 지나면 Normal로 복귀해야 정상

**통과 기준**:
- [ ] 누운 직후 5~10초 동안은 lying 표시 OK
- [ ] 1분 이상 누워 있어도 알람 반복 안 됨
- [ ] 호흡으로 인한 미세 움직임이 화면 `micro_motion` 값에 반영됨 (5 이상)

⚠️ 만약 누워있는 동안 ACCIDENT 알람이 다시 울리면 → 자는 사람도 사고로 오인. 미세 움직임 임계값 조정 필요.

---

## 시나리오 4 — 의자에 천천히 앉기 (정상)

**동작**: 서있다가 → 천천히 의자에 앉음 → 정자세로 30초 유지

**예상**:
- 자세: `standing` → `sitting`
- 점수: sitting 자세에서 0~2점 (sitting 임계값 5점이므로 미달)
- 라벨: `Normal` 유지

**통과 기준**:
- [ ] 앉는 동안 FALL 알람 안 울림
- [ ] 30초 정지 동안 알람 안 울림 (sitting + 정상 자세 → accident_by_stillness 발동 X)
- [ ] 자세 표시가 `sitting (OK)` 또는 `standing(OK)`

---

## 시나리오 5 — 의자에서 졸기 / 고개 숙임 (사고 의심)

**동작**: 의자에 앉아 → 고개를 천천히 깊게 푹 숙임 (코가 어깨선 아래로) → 5초 이상 유지

**예상**:
- 자세: `sitting` + `ABNORMAL` 표시
- 5초 이상 비정상 자세 지속 → score 가산
- `accident_by_stillness` 발동 가능 (`abnormal + stillness >= 5초`)
- 라벨: 5~10초 후 `ACCIDENT!` 표시

**통과 기준**:
- [ ] 고개 숙인 직후 자세 라벨에 `ABNORMAL` 표시
- [ ] 5초 후 ACCIDENT 알람 (의식 잃은 사람 시뮬레이션)
- [ ] 고개를 다시 들면 Normal로 복귀

---

## 시나리오 6 — 카메라 가까이 접근

**동작**: 카메라에서 매우 가까이 접근 (어깨가 화면 폭의 35% 이상 차지하도록) → 그 상태로 일상 동작

**예상**:
- 자세 라벨에 `[TOO CLOSE]` 표시
- 가까울 때는 lying 임계값을 0.55→0.70으로 올려 일상 자세를 lying으로 오인하지 않음

**통과 기준**:
- [ ] `[TOO CLOSE]` 라벨 표시
- [ ] 가까이서 일상 자세(앉기/서기/팔 움직이기)에서 FALL 오탐 없음

---

## 시나리오 7 — 부분 가림 (하반신 안 보임)

**동작**: 책상이나 가구로 하반신을 가린 채로 → 의자에 앉아있다가 일어남

**예상**:
- knee visibility 낮음
- 자세: `unknown` 또는 `standing` (어깨-엉덩이 차이로 판단)
- 화면에 `knee_visible: False` 표시 (백엔드 로그)

**통과 기준**:
- [ ] 하반신 가려도 시스템 크래시 없음
- [ ] knee 안 보이는 상태에서 일상 동작 시 FALL 오탐 없음
- [ ] (선택) 누우면 → 어깨-엉덩이 y차가 작아져 lying 판정 가능 (classify_posture 개선 효과 확인)

---

## 시나리오 8 — 누워서 폰 보기 (정상)

**동작**: 카메라 켜진 채로 → 매트/소파에 누워서 1분간 폰 보기 (가끔 손 움직이기)

**예상**:
- 카메라 시작 시점부터 lying이라면 `last_upright_time = None` → lying 점수 -1 패널티
- 또는 처음에 서있다가 누우면 → 자세 전환 보너스 발동 가능하지만 미세 움직임/정상 자세로 사고 아님 처리
- 라벨: Normal 또는 짧게 FALL 후 Normal 복귀

**통과 기준**:
- [ ] 누워있는 1분 동안 알람 반복 X
- [ ] 만약 처음에 짧게 FALL이 떴어도 곧 Normal로 복귀
- [ ] 폰 보면서 손/얼굴 움직임이 미세 움직임 카운터에 반영

---

## 시나리오 9 — 운동 / 빠른 움직임 (정상)

**동작**: 가벼운 스트레칭, 팔 흔들기, 제자리 뛰기 등 (낙상 아님)

**예상**:
- 자세: standing 유지
- 빠른 움직임으로 velocity가 일시적으로 튈 수 있음
- 0.5초 지속 필터로 한 프레임 튐 차단

**통과 기준**:
- [ ] 빠른 동작 중 FALL 오탐 없음
- [ ] 자세 라벨이 standing/sitting로 안정적

---

## 시나리오 10 — 어두운 조명

**동작**: 방 조명을 절반 또는 그 이상 어둡게 → 일상 동작 + 1회 낙상

**예상**:
- CLAHE 밝기 보정 작동 (5프레임마다)
- landmark 정확도가 떨어질 수 있음
- 그래도 명확한 낙상은 잡혀야 함

**통과 기준**:
- [ ] 어두운 환경에서도 정상 자세는 인식
- [ ] 명확한 낙상 1회는 감지됨 (몇 프레임 지연 허용)

---

## 시나리오 11 — 여러 명 등장

**동작**: 두 명이 동시에 화면에 → 한 명만 쓰러짐

**예상**:
- MediaPipe Pose는 1명만 추적 (기본 설정)
- 가장 큰 사람 또는 신뢰도 높은 사람 기준으로 판정

**통과 기준**:
- [ ] 시스템 크래시 없음
- [ ] 적어도 한 명의 낙상은 감지

⚠️ 다중 인원 추적은 본 시스템의 한계입니다 (단일 인물 가정).

---

## 시나리오 12 — 30초 쿨다운 검증

**동작**: 시나리오 1처럼 1차 낙상 → 30초 이내에 다시 일어났다 → 또 쓰러짐

**예상**:
- 1차 캡처는 저장됨
- 2차 낙상은 알람은 울리지만 캡처는 30초 쿨다운으로 차단

**통과 기준**:
- [ ] 첫 캡처 시각 후 30초 이내에 추가 캡처 없음
- [ ] 30초 후에 일어난 3차 낙상은 새 캡처로 저장됨

---

## 결과 기록 양식

각 시나리오마다 다음을 기록:

```
시나리오 N: ✅ / ❌
- 감지 시간: ___초
- 점수: MP=__ YOLO=__ MAE=__
- 라벨: ___
- 자세: ___
- 메모: ___
```

## 종합 통과 기준

- **민감도 측면 (낙상은 무조건 잡혀야 함)**: 시나리오 1, 2, 5, 10 → 4개 모두 통과
- **특이도 측면 (정상은 알람 X)**: 시나리오 3, 4, 8, 9 → 4개 모두 통과
- **로버스트성**: 시나리오 6, 7, 11, 12 → 3개 이상 통과

12개 중 10개 이상 통과 시 → 운영 신뢰도 충분.
8~9개 → 추가 튜닝 필요.
7개 이하 → 옵션 B 진단 결과 재검토 후 추가 보강.
